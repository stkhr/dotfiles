# 外部API・クラウド設定の事前確認

コードを書く前に、必ず一次情報(公式ドキュメント or 実機のCLI/APIレスポンス)で以下2軸を確認する:

- **識別子と命名規則**: action prefix・サービス識別子・suffix・正規表現制約
- **スキーマと制約値**: リクエスト/レスポンス構造・必須フィールド・プラットフォーム推奨設定との整合

「実装 → 実行時エラー → 段階的修正」はユーザーのレビュー負荷を増やす。実装着手前の確認を優先する。
