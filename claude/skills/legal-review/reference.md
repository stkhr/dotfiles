# 法令別トリガー逆引きリファレンス

legal-review スキルの知識部。「こういう仕様・要件が出たら、この法令を疑う」という
逆引き表で構成する。数値基準・施行状況は執筆時点(2026年)のスナップショットのため、
アラート検知後に必ず一次情報(各省庁サイト)で裏取りすること。

## 1. 資金決済法(前払式支払手段)

ユーザーが先にお金を払い、後からサービス内で使う残高・ポイントが対象。

トリガー:

- ポイント・残高を発行する(購入特典/チャージ/キャンペーンのおまけ付与)
- ポイント・残高を商品・サービスの代金に充当できる
- ポイントの有効期限を設定・変更する
- 他社サービス・加盟店でもポイントを使えるようにする
- 退会時に残高を現金で返金する

確認事項・分岐条件:

| 分岐 | 内容 |
|------|------|
| 該当要件 | 金額・数量の記録 / 対価を得た発行 / 代価弁済への使用可、の 3 要件(第3条)。購入特典でも対価性が認められうる。無償のおまけ分もチャージと一体で対価性ありと解釈されるのが一般的 |
| 有効期限 | 発行から 6 ヶ月以内に限り使用できるものは適用除外(第4条)。1 日でも超えると適用対象。期限設定は供託コストとユーザー体験のトレードオフでありビジネス判断を含む |
| 自家型 | 自サービス内でのみ使用可。基準日(3月末・9月末)の未使用残高が 1,000 万円を超えたら事後届出+半額以上の供託(第14条)。残高総量を監視するクエリ/バッチを早期に用意する |
| 第三者型 | 他社加盟店でも使用可。**サービス開始前の事前登録**が必要で審査に数ヶ月かかる。自家型→第三者型への仕様変更はリリース日程を覆すため、要件が出た時点でアラート |
| 払い戻し | 現金払い戻しは原則禁止(第20条)。「退会時に残高返金」という仕様はこの規定に反しないか確認 |

## 2. 資金決済法(資金移動業)

ユーザーから受け取ったお金を他者へ移転する機能が対象。無登録での資金移動業は違法。

トリガー:

- ユーザー間でポイント・金額を送り合える
- クリエイターへの投げ銭・支援機能
- ポイントを現金・ギフト券・暗号資産などに換金できる(将来対応の要望を含む)

確認事項・分岐条件:

| 分岐 | 内容 |
|------|------|
| 換金性 | 最重要の分岐点。受け取った側が現金・ギフト券等に換えられるなら「ポイントだから大丈夫」は通らず、実質的な送金とみなされうる |
| 区分 | 送金上限で第一種(100万円超)/第二種(100万円以下)/第三種(1万円以下)。いずれも金融庁への登録が必要 |
| 対価スキーム | 「ユーザー→プラットフォームへの機能購入・コンテンツ購入」に組み替え、支払いに対価(コンテンツ・機能・権利)を設定する設計は登録なしで成立している例がある。ただし対価の実態が薄いと実質的な資金移動とみなされるリスクがあり、必ず専門家確認 |

## 3. 電気通信事業法

他人の通信を媒介する機能と、利用者情報の外部送信が対象。2023年6月施行の改正で
一般の Web サービスにも適用範囲が広がった。

トリガー:

- ユーザー同士のメッセージ・チャット・DM 機能
- 非公開グループ・鍵つきコミュニティ内のやりとり
- 管理者がユーザーのメッセージ内容を検索・閲覧できる機能(通報対応・サポート含む)
- アナリティクスタグ・広告 SDK・SNS ボタンなど外部スクリプトの埋め込み

確認事項・分岐条件:

| 分岐 | 内容 |
|------|------|
| 届出要否 | メッセージ機能が「主たる目的」(SNS・チャットアプリ)なら届出が必要な可能性が高い。EC の問い合わせチャットのような「付随的機能」なら不要とされる場合がある。機能の位置づけが変わると要否も変わる。判断は総務省ガイドライン+専門家確認 |
| 通信の秘密 | 通信の内容・存在・当事者をみだりに第三者に知らせてはならない(第4条)。管理者の自由閲覧は業務上の理由(監視・サポート)があっても侵害になりうる。規約末尾の包括同意では不十分とする見解が有力で、個別の明確な同意や「通報時のみ閲覧可」等の手続き設計が必要 |
| 設計指針 | 「原則閲覧不可・例外時のみ手続きを経て閲覧可」をシステムのアクセス制御として実装しておくと法的に説明しやすい |
| 外部送信規律 | タグ・SDK の埋め込みは「利用者情報を外部に送信する仕組みの設置」。送信先・送信内容・目的の通知または公表(プライバシーポリシー等への記載)が義務。「タグを入れるだけ」の作業にも確認が必要 |

## 4. 個人情報保護法

トリガー:

- 退会機能の実装(論理削除/物理削除の選択)
- 退会後もデータを分析目的で保持する
- 分析ツール・CRM・メール配信等の外部サービスへユーザーデータを連携する
- サードパーティ SDK・スクリプトの埋め込み(行動データの外部送信)
- ユーザーデータの海外サーバーへの保存・転送

確認事項・分岐条件:

| 分岐 | 内容 |
|------|------|
| 消去請求 | 本人は利用停止・消去を請求できる(第35条)。「保有の必要がなくなった場合」も対象。退会済みユーザーのデータは該当と解釈されうる |
| 論理削除 | deleted_at フラグだけでは「消去した」と言えない。求められるのは「特定の個人を識別できなくすること」であり、識別子(氏名・メール・住所等)のマスキング+属性データの統計用分離が現実的な設計 |
| 保存義務との分離 | 取引記録は電子帳簿保存法等で保存義務がある(退会後も保持する正当理由)。「消すべき個人情報」と「残すべき法定保存データ」をテーブル/ストレージレベルで分離する設計にする |
| 第三者提供 | 本人の同意なき第三者提供は原則禁止(第27条)。外部 API に個人情報を送る行為が該当。対応は (a) ポリシー明記+同意取得、(b) 委託構成(委託先の監督義務が発生)のいずれか。デバッグログ・テストデータ経由の意図しない送信にも注意 |
| 海外移転 | 外国にある第三者への提供は移転先国名等の公表・同意が必要 |

## 5. 著作権法(素材・ライセンス・コンテンツ)

トリガー:

- フリー素材・有料素材・OSS ライブラリを新たに使う
- GPL / AGPL 系ライセンスのコードをプロダクトに組み込む
- フォントをサーバー配信・PDF 埋め込み・アプリ同梱する
- ユーザー投稿コンテンツ(UGC)を広告・プロモーションに使う
- AI 生成コンテンツを素材として使う
- 他サービスのデザイン・UI をそのまま再現する

確認事項・分岐条件:

| 分岐 | 内容 |
|------|------|
| ライセンス種別 | 「無料=自由に使える」ではない。CC BY はクレジット表記が条件、CC0 は不要、と同じ CC でも全く異なる。表記漏れは著作権侵害 |
| コピーレフト | GPL / AGPL のコードを組み込むとプロダクト全体に同ライセンスでの公開義務が及びうる。CI で依存ライセンス一覧を検査する体制(license-checker 系ツール)が理想 |
| フォント | デスクトップ利用・Web 配信・PDF 埋め込み・アプリ同梱は別ライセンスのことが多い。PDF 出力機能やフォント同梱アプリは特に注意 |
| UGC | 投稿コンテンツの著作権は投稿者に帰属。広告利用には規約での非独占ライセンス条項または個別許諾が必要。規約があっても想定外の利用(他社への素材提供等)は範囲外とみなされうる |
| AI 生成物 | 生成物の著作権帰属は法的に未確定。特定作家のスタイル模倣は元著作物への依拠とみなされるリスク。権利状態の確認責任は利用者側にある |

## 6. 特定商取引法(購入・解約フロー)

トリガー:

- EC・通販の購入フローを実装する(単品購入も対象)
- 定期購入・サブスクリプションの購入フローを実装する
- 「初回○○円」等のキャンペーン表示を実装する
- 解約フローの UI を設計する
- 特商法に基づく表記ページの新設・更新

確認事項・分岐条件:

| 分岐 | 内容 |
|------|------|
| 最終確認画面 | 購入ボタン直前の画面に、商品内容・販売価格(送料含む総額)・支払時期と方法・撤回/解約条件の明示が義務。通販全般に適用 |
| 定期購入 | 2022年改正で強化。定期購入である旨・継続分を含む支払総額・解約方法/条件の表示が追加で必須。初回価格だけ目立たせ継続金額を小さく表示する UI は「不当な表示」で行政処分の対象 |
| 解約 UI | 解約を不当に困難にする UI(隠された解約ボタン・繰り返しの引き止め誘導・多段アンケート)は違法なダークパターン。解約完了までのフロー全体をレビュー対象にする |
| 表記ページ | 事業者名・所在地・連絡先・価格・支払方法・返品/解約条件の表記ページが必要。存在と情報の鮮度を確認 |
| 実装者の立場 | 表示内容・文字サイズ・ボタン配置はフロントエンド実装そのもの。「デザイン指示通りに作った」でも法的要件の確認はできる立場にある |

## 7. スクレイピング・データ収集

トリガー:

- 外部サービスのデータを自動収集して自社サービスに取り込む
- ログインが必要なページへのアクセスを自動化する
- 競合の商品・価格情報を定期収集する
- 短時間に大量リクエストを送る処理を実装する

確認事項・分岐条件:

| 分岐 | 内容 |
|------|------|
| 規約と法律の区別 | robots.txt に法的拘束力はなく、利用規約違反は民事(アカウント停止・損害賠償)。不正アクセス禁止法・不正競争防止法違反は刑事。両者を混同しない |
| 認証の壁 | 認証を回避したアクセス、他人の認証情報でのアクセスは不正アクセス禁止法違反(刑事罰) |
| データの体系的複製 | DB 内容の体系的な大量取得やそれを元にした同種サービス構築は不正競争防止法(限定提供データ等)のリスク |
| 過負荷 | 悪意がなくても過度なリクエストで相手サービスを停止させれば電子計算機損壊等業務妨害になりうる(実際に逮捕例がある)。リクエスト間隔・並列数の制限、User-Agent への連絡先記載は法的防御策として必須 |

## 8. 販売商材別早見表

「何を売るか」で追加の許認可・規制が発生する。許認可は申請に数週間〜数ヶ月かかるため、
該当商材があれば実装前にアラートを出す。代表例であり網羅ではない。

| 商材 | 主な法令 | アラートポイント |
|------|---------|-----------------|
| 酒類 | 酒税法 | 酒類販売業免許。年齢確認 UI の実装義務 |
| 化粧品・石鹸 | 薬機法 | 効能効果の広告表現規制。ユーザーレビュー表示も運営側の違反になりうる |
| 健康食品・サプリ | 薬機法・食品表示法 | 医薬品的表現は違法。機能性表示食品は届出制。アフィリエイト・SNS 投稿も規制対象 |
| 食品・飲食 | 食品衛生法・食品表示法 | 営業許可(保健所)。アレルゲン・原材料表示 |
| 中古品売買 | 古物営業法 | 買い取りには古物商許可。オンライン本人確認(eKYC)の実装が必要になりやすい |
| ジム・エステ・語学教室 | 特商法・割賦販売法 | 継続的役務提供の中途解約・返金規制。契約時書面の交付義務 |
| チケット転売 | チケット不正転売禁止法 | 興行チケットの定価超え転売は刑事罰。出品者本人確認の検討 |
| 懸賞・ガチャ | 景品表示法 | 景品の価格上限。ガチャ確率の正確な表示義務。二重価格表示の禁止 |
| マッチング・出会い系 | 出会い系サイト規制法 | 18歳未満利用禁止(年齢確認の実装)。異性紹介事業の届出 |
| 不動産 | 宅建業法 | 売買・賃貸仲介には宅建業免許 |
| 旅行・ツアー | 旅行業法 | 販売・手配には旅行業登録 |
| 金融商品・投資 | 金融商品取引法 | 投資助言・代理業等の登録 |
| 医療・歯科 | 医療法・薬機法 | 広告規制。「治療」「医療」等の表現制約 |
