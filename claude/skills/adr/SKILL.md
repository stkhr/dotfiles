# ADR (Architecture Decision Record) Skill

## Purpose

技術的な意思決定を構造化されたADR形式で記録します。
**ステータスの明示**が最重要です。AIエージェントは廃止されたADRを現行ルールと誤認するため、
すべてのADRに明確なステータスを付与します。

## ADR ステータス

| ステータス | 意味 |
|-----------|------|
| `Proposed` | 提案中・未確定 |
| `Accepted` | 承認済み・現行ルール |
| `Superseded by ADR-XXX` | ADR-XXXに置き換えられた（廃止） |
| `Deprecated` | 理由なく廃止 |
| `Rejected` | 却下 |

## ファイル命名規則

```
docs/adr/ADR-NNN-short-title.md
```

NNNは3桁ゼロ埋め（例: `ADR-001`, `ADR-042`）

## ADR テンプレート

```markdown
# ADR-NNN: タイトル

**ステータス**: Accepted
**日付**: YYYY-MM-DD
**決定者**: （名前 or チーム名）

## コンテキスト

この決定が必要になった背景と制約を記述する。
なぜこの問題を解決する必要があったか。

## 決定

何をすることにしたか、一文で明確に述べる。

## 理由

この選択肢を選んだ理由。検討した代替案とその却下理由も含める。

### 検討した代替案

- **Option A**: ～ → 却下理由: ～
- **Option B**: ～ → 却下理由: ～

## 結果

### ポジティブな影響
- ～

### ネガティブな影響・トレードオフ
- ～

## 関連

- 置き換えるADR: ADR-XXX（ある場合）
- 関連ADR: ADR-YYY
```

## 実行手順

1. **既存ADRの確認**: `docs/adr/` ディレクトリを確認し、次の番号を特定
2. **内容の確認**: ユーザーに以下を確認
   - 決定の内容（What）
   - 決定の理由（Why）
   - 検討した代替案
3. **ADR作成**: テンプレートに従いファイルを作成
4. **既存ADRの更新**: 置き換える場合は対象ADRのステータスを `Superseded by ADR-NNN` に更新
5. **CLAUDE.mdへの参照追加**: プロジェクトの `CLAUDE.md` に ADR ディレクトリへのポインターがなければ追加

## 重要な原則

- **Howのみ変更可**: ADRが記録するのは実現方法（How）の決定。What（目的）は変えない
- **リンクで腐敗を検出可能に**: 外部ファイルへのリンクは壊れると検出できるが、散文は静かに腐敗する
- **廃止ADRには必ず後継を明記**: `Superseded by ADR-NNN` として後継ADRを示す
