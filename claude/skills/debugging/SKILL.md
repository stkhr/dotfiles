# 障害調査・デバッグ

- 原因特定が先。原因が確定するまで修正案に進まない。アプリ側・インフラ側の両方の変更履歴を洗う
- 同じ問題で仮説が3回外れたら、レイヤを1つ上げて疑う(ミドルウェア → アプリ → DB / ネットワーク → OS)
- 推測のまま設定値・コード値を確定させない。実機検証(CLIワンライナーをユーザーに提供して結果を貰う等)を挟む
