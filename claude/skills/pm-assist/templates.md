# PM Assist Templates — アウトプット雛形

Step 6 で使うアウトプットの雛形。形態（軽量型 / エンプラ型）に応じて選ぶ。
セクションの追加・削除は自由だが、**リスク・不確実性セクションだけは削除禁止**。

## 軽量型テンプレート

```markdown
# {プロジェクト名} 整理メモ（YYYY-MM-DD）

## 現状サマリー

- **フェーズ**: 計画（判断根拠: ゴールは合意済み、タスク分解が未着手）
- **形態**: 軽量型（3人 / 2ヶ月 / ステークホルダーは直属の上長のみ）
- **ゴール**: {完了の定義を1〜2文で}
- **期限**: {日付}（{ハード / 目安}）

## タスク一覧

| # | タスク | 担当 | 依存 | 優先度 | 備考 |
|---|--------|------|------|--------|------|
| 1 | ... | 田中 | - | 高 | |
| 2 | ... | 未定 | #1 | 高 | クリティカルパス |
| 3 | ... | 佐藤 | - | 中 | |

**マイルストーン**: {日付}: {何ができている状態か}

## リスク・不確実性（必須セクション）

### 決まっていない・分かっていないこと
- {不明点} → 対応: {調査タスク / 誰にいつ確認するか}

### 前提にしていること
- {前提} → 崩れた場合: {影響}。検証: {誰が・いつ}

### リスク トップ5
| リスク | 確率×影響 | 対応方針 | 次のアクション |
|--------|----------|---------|---------------|
| ... | 高×高 | 軽減 | ... |

## 直近のネクストアクション（今週〜2週間）

1. [ ] ...
2. [ ] ...
3. [ ] ...

## PMの判断が必要な事項

- **Q1**: {質問}。推奨: {案}（理由: ...）
- **Q2**: ...
```

## エンプラ型テンプレート

- **ゴール**: {完了の定義}
